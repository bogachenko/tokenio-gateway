  455: 	if filter.CreatedTo != nil {

  457: 	}

  459: 		return "", args
  460: 	}

   50: # 2. Главный routing invariant
   51: 
   52: Route selection не должен выполняться только по `model`.
   53: 
   54: Правильный routing key:
   55: 
   56: ```text
   57: api_family + endpoint_kind + client_model
   58: ```
   59: 
   60: Fallback разрешён только между routes с одинаковыми:
   61: 
   62: ```text
   63: api_family
   64: endpoint_kind
   65: client_model
   66: ```
   67: 
   68: Запрещено:
   69: 
   70: ```text
   71: OpenAI-compatible request -> Gemini-native route
   72: Gemini-native request -> OpenAI-compatible route
   73: Anthropic-native request -> OpenAI-compatible route
   74: Ollama-native request -> OpenAI-compatible route
   75: ```
   76: 
   77: Gateway не конвертирует semantic request payload между API families; explicit model identifier rewrite не меняет API family.
   78: 
   79: ---
   80: 
   81: # 3. Provider type
   82: 
   83: `provider_type` — тип upstream/provider ecosystem.
   84: 
   85: Поддерживаемые значения:
   86: 
   87: ```text
   88: openai
   89: openrouter
   90: together
   91: groq
   92: ollama
   93: lmstudio
   94: vllm
   95: gemini
   96: anthropic
   97: hydra
   98: ```
   99: 
  100: `provider_type` не является reseller.
  101: 
  102: Пример:
  103: 
  104: ```text
  105: provider_type = openrouter
  106: ```
  107: 
  108: означает класс upstream/provider, но не конкретный аккаунт.
  109: 
  110: ---
  111: 
  112: # 4. Reseller
  113: 
  114: ## 4.1. Определение
  115: 
  116: Reseller — конкретный upstream account/base_url/API key/balance/limits.
  117: 
  118: Reseller содержит:
  119: 
  120: ```text
  121: id
  122: name
  123: provider_type
  124: base_url
  125: api_key_env
  126: enabled
  127: balance_cents
  128: reserved_cents
  129: minimum_balance_cents
  130: created_at
  131: updated_at
  132: ```
  133: 
  134: ## 4.2. Reseller API key
  135: 
  136: Reseller API key хранится не в БД, а в environment variable.
  137: 
  138: В БД хранится только имя env-переменной:
  139: 
  140: ```text
  141: api_key_env
  142: ```
  143: 
  144: Пример:
  145: 
  146: ```text
  147: api_key_env = OPENROUTER_PRIMARY_API_KEY
  148: ```
  149: 
  150: Gateway читает секрет так:
  151: 
  152: ```text
  153: os.Getenv(route.reseller.api_key_env)
  154: ```
  155: 
  156: Если env отсутствует или пустой, все routes этого reseller считаются unavailable.
  157: 
  158: Причина unavailable:
  159: 
  160: ```text
  161: missing_reseller_api_key
  162: ```
  163: 
  164: Обычному client response нельзя раскрывать имя env-переменной.
  165: 
  166: ## 4.3. Reseller enabled flag
  167: 
  168: Если:
  169: 
  170: ```text
  171: reseller.enabled = false
  172: ```
  173: 
  174: все routes reseller считаются unavailable.
  175: 
  176: Причина:
  177: 
  178: ```text
  179: manual_disabled
  180: ```
  181: 
  182: ## 4.4. Reseller balance
  183: 
  184: Gateway хранит расчётный баланс reseller в RUB cents:
  185: 
  186: ```text
  187: balance_cents
  188: reserved_cents
  189: minimum_balance_cents
  190: ```
  191: 
  192: `balance_cents` — расчётный остаток средств у reseller.
  193: `reserved_cents` — сумма preflight reserves по in-flight запросам.
  194: `minimum_balance_cents` — минимальный остаток, ниже которого reseller не должен использоваться.
  195: 
  196: Доступный баланс reseller:
  197: 
  198: ```text
  199: available_reseller_balance_cents =
  200:   balance_cents - reserved_cents - minimum_balance_cents
  201: ```
  202: 
  203: Если:
  204: 
  205: ```text
  206: available_reseller_balance_cents < estimated_upstream_cost_cents
  207: ```
  208: 
  209: route недоступен.
  210: 
  211: Причина:
  212: 
  213: ```text
  214: insufficient_reseller_balance
  215: ```
  216: 
  217: ---
  218: 
  219: # 5. Route
  220: 
  221: ## 5.1. Определение
  222: 
  223: Route — возможность купить конкретную client model у конкретного reseller через конкретный API family и endpoint kind.
  224: 
  225: Route содержит:
  226: 
  227: ```text
  228: id
  229: reseller_id
  230: provider_type
  231: api_family
  232: endpoint_kind
  233: client_model
  234: provider_model
  235: model_rewrite_policy
  236: enabled
  237: priority
  238: requests_per_minute
  239: tokens_per_minute
  240: concurrent_requests
  241: default_max_output_tokens
  242: capabilities
  243: cooldown_until
  244: cooldown_reason
  245: created_at
  246: updated_at
  247: ```
  248: 
  249: ## 5.2. Route identity
  250: 
  251: Route identity должна быть стабильной.
  252: 
  253: Route ID используется в:
  254: 
  255: ```text
  256: usage ledger
  257: reseller balance reserve
  258: debug/admin API
  259: cooldown
  260: logs
  261: metrics
  262: ```
  263: 
  264: Route ID не должен быть публичным client-facing identifier.
  265: 
  266: ## 5.3. Client model
  267: 
  268: `client_model` — имя модели, которое видит клиент.
  269: 
  270: Пример:
  271: 
  272: ```text
  273: gpt-4.1-mini
  274: gemini-2.5-flash
  275: claude-3-5-sonnet
  276: ```
  277: 
  278: Client request использует именно `client_model`.
  279: 
  280: ## 5.4. Provider model
  281: 
  282: `provider_model` — имя модели, которое нужно отправить upstream reseller.
  283: 
  284: Пример:
  285: 
  286: ```text
  287: client_model: gpt-4.1-mini
  288: provider_model: openai/gpt-4.1-mini
  289: ```
  290: 
  291: Если API family и reseller позволяют использовать client model без изменения, тогда:
  292: 
  293: ```text
  294: provider_model = client_model
  295: model_rewrite_policy = none
  296: ```
  297: 
  298: Если upstream reseller требует другое имя модели, route обязан явно включить:
  299: 
  300: ```text
  301: model_rewrite_policy = provider_model
  302: ```
  303: 
  304: В этом случае provider adapter может заменить только model identifier.
  305: 
  306: Для OpenAI-compatible requests разрешённая mutation ограничена:
  307: 
  308: ```text
  309: body.model = route.provider_model
  310: ```
  311: 
  312: Для path-based native APIs разрешённая mutation ограничена model path segment, если adapter явно поддерживает эту операцию.
  313: 
  314: Это не является API-format conversion.
  315: 
  316: Запрещено вместе с model rewrite:
  317: 
  318: ```text
  319: конвертировать messages/content/tools
  320: конвертировать multimodal payload
  321: конвертировать response_format
  322: менять semantic request payload
  323: делать fallback в другую API family
  324: ```
  325: 
  326: Если `model_rewrite_policy = provider_model`, но adapter не поддерживает безопасную model rewrite для данного API family, route считается invalid и не должен выбираться.
  327: 
  328: Model rewrite route selection rule:
  329: 
  330: ```text
  331: routes requiring model_rewrite_policy = provider_model are eligible only if adapter supports safe model identifier rewrite for this api_family.
  332: ```
  333: 
  334: If adapter does not support safe model rewrite:
  335: 
  336: ```text
  337: route is treated as unavailable
  338: reason = unsupported_model_rewrite_policy
  339: ```
  340: 
  341: 
  342: 
  343: 
  344: ## 5.5. Model rewrite policy
  345: 
  346: Allowed values:
  347: 
  348: ```text
  349: none
  350: provider_model
  351: ```
  352: 
  353: `none` означает:
  354: 
  355: ```text
  356: client model identifier отправляется upstream без изменения
  357: ```
  358: 
  359: `provider_model` означает:
  360: 
  361: ```text
  362: adapter заменяет только model identifier на route.provider_model
  363: ```
  364: 
  365: Route validation:
  366: 
  367: ```text
  368: if provider_model != client_model, model_rewrite_policy must be provider_model
  369: if model_rewrite_policy = provider_model, provider_model must be non-empty
  370: if model_rewrite_policy = none, provider_model should equal client_model unless adapter explicitly documents otherwise
  371: ```
  372:

  381: 
  382: ```text
  383: openai_compatible
  384: gemini_native

  386: ollama_native

  392: openai_compatible
  393: ```

  396: 
  397: Gateway определяет `api_family` по endpoint path.
  398:

  402: /v1/chat/completions    -> openai_compatible
  403: /v1/embeddings          -> openai_compatible
  404: /v1/images/generations  -> openai_compatible
  405: /v1/models              -> openai_compatible

  407:

  411: 
  412: Fallback между routes возможен только внутри одного API family.
  413: 
  414: Запрещено fallback-ить между:
  415: 
  416: ```text
  417: openai_compatible <-> gemini_native
  418: openai_compatible <-> anthropic_native
  419: openai_compatible <-> ollama_native
  420: gemini_native <-> anthropic_native
  421: ```

  426: 
  427: Поддерживаемые endpoint kinds первой версии:
  428: 
  429: ```text
  430: chat
  431: embeddings
  432: images_generation
  433: models
  434: health
  435: ```
  436: 
  437: Routing применяется только к LLM upstream endpoints:
  438: 
  439: ```text
  440: chat
  441: embeddings
  442: images_generation
  443: ```
  444: 
  445: `models` строится из registry/routes, но не forward-ится как обычный LLM request.
  446:

  449: Фильтрация только enabled routes в persistence layer запрещена:
  450: application layer должен видеть полный registry state, чтобы

  452: 
  453: `health` не использует routing.

  457: # 8. Capabilities
  458: 
  459: ## 8.1. Capability set
  460: 
  461: Route должен объявлять capabilities.
  462: 
  463: Минимальный набор:
  464: 
  465: ```text
  466: chat
  467: embeddings
  468: images_generation
  469: tools
  470: tool_choice
  471: response_format
  472: json_schema
  473: image_input
  474: audio_input
  475: file_input
  476: video_input
  477: reasoning
  478: ```
  479: 
  480: ## 8.2. Capability validation
  481: 
  482: Gateway должен определить requested capabilities структурно из request body.
  483: 
  484: Запрещено определять capabilities по смыслу prompt text.
  485: 
  486: Примеры structural detection:
  487: 
  488: ```text
  489: tools present             -> tools
  490: tool_choice present       -> tool_choice
  491: response_format present   -> response_format
  492: response_format json_schema -> json_schema
  493: image content present     -> image_input
  494: audio content present     -> audio_input
  495: file/document present     -> file_input
  496: video content present     -> video_input
  497: reasoning_effort present  -> reasoning
  498: ```
  499: 
  500: Если request требует capability, которой нет у route, route исключается из candidate set.
  501: 
  502: Если model существует, но нет route с нужными capabilities:
  503: 
  504: ```text
  505: HTTP 400
  506: error.code = unsupported_capability
  507: ```
  508:

  511: # 9. Route price
  512: 
  513: ## 9.1. Route-specific price
  514: 
  515: Цена задаётся на уровне route.
  516: 
  517: Route price содержит:
  518: 
  519: ```text
  520: route_id
  521: currency
  522: input_price_per_1m_tokens_cents
  523: cached_input_price_per_1m_tokens_cents
  524: output_price_per_1m_tokens_cents
  525: reasoning_output_price_per_1m_tokens_cents
  526: image_input_price_per_1m_tokens_cents
  527: audio_input_price_per_1m_tokens_cents
  528: audio_output_price_per_1m_tokens_cents
  529: file_input_price_per_1m_tokens_cents
  530: video_input_price_per_1m_tokens_cents
  531: image_generation_price_per_unit_cents
  532: image_generation_unit_kind
  533: markup_coefficient
  534: ```
  535: 
  536: Currency первой версии:
  537: 
  538: ```text
  539: RUB
  540: ```
  541: 
  542: ## 9.2. Selection cost
  543: 
  544: Route selection сортирует candidates по estimated total upstream cost, а не по одной input price.
  545: 
  546: Estimated total upstream cost рассчитывается до markup.
  547: 
  548: Client sell price рассчитывается отдельно pricing layer.
  549: 
  550: ## 9.3. Markup
  551: 
  552: Markup не влияет на выбор самого дешёвого upstream route, если markup одинаковый.
  553: 
  554: Если markup задан per route, route selection для клиента может учитывать estimated client amount.
  555: 
  556: Решение первой версии:
  557: 
  558: ```text
  559: route selection сортирует по estimated_upstream_cost_cents
  560: billing для клиента считает selected route price * markup
  561: ```
  562: 
  563: ---
  564: 
  565: # 10. Route availability
  566: 
  567: Route считается available, если выполняются все условия:
  568: 
  569: ```text
  570: route.enabled = true
  571: reseller.enabled = true
  572: reseller api key env exists and non-empty
  573: route cooldown is absent or expired
  574: required capabilities are supported
  575: estimated upstream cost <= available reseller balance
  576: route rate/concurrency limits allow request
  577: route model_rewrite_policy is supported by adapter
  578: ```
  579: 
  580: Если хотя бы одно условие не выполнено, route исключается из candidates.
  581: 
  582: ---
  583: 
  584: # 11. Route selection algorithm
  585: 
  586: ## 11.1. Inputs
  587: 
  588: Route selector получает:
  589: 
  590: ```text
  591: user_id
  592: api_family
  593: endpoint_kind
  594: client_model
  595: requested_capabilities
  596: estimated_usage
  597: request_id
  598: idempotency_key
  599: now
  600: ```
  601: 
  602: ## 11.2. Algorithm
  603: 
  604: Route selector должен выполнить:
  605: 
  606: ```text
  607: 1. Найти enabled routes для:
  608:    api_family + endpoint_kind + client_model
  609: 
  610: 2. Если routes не найдены:
  611:    return unknown_model
  612: 
  613: 3. Отфильтровать routes, чей reseller disabled.
  614: 
  615: 4. Отфильтровать routes с missing reseller API key.
  616: 
  617: 5. Отфильтровать routes в active cooldown.
  618: 
  619: 6. Отфильтровать routes без required capabilities.
  620: 
  621: 7. Рассчитать estimated_upstream_cost_cents для каждого candidate.
  622: 
  623: 8. Отфильтровать routes, где:
  624:    available_reseller_balance_cents < estimated_upstream_cost_cents
  625: 
  626: 9. Отфильтровать routes по локальным rate/concurrency limits.
  627: 
  628: 10. Отфильтровать routes, где:
  629:     model_rewrite_policy требует rewrite,
  630:     но adapter не поддерживает safe model identifier rewrite для этого api_family.
  631: 
  632: 11. Отсортировать candidates по:
  633:     estimated_upstream_cost_cents ASC
  634:     priority ASC
  635:     route_id ASC
  636: 
  637: 12. Выбрать первый candidate.
  638: 
  639: 13. Зарезервировать estimated_upstream_cost_cents на reseller.
  640: 
  641: 14. Вернуть selected route.
  642: ```
  643: 
  644: ## 11.3. Unknown model
  645: 
  646: Если route registry не содержит ни одного route для:
  647: 
  648: ```text
  649: api_family + endpoint_kind + client_model
  650: ```
  651: 
  652: ответ:
  653: 
  654: ```text
  655: HTTP 400
  656: error.code = unknown_model
  657: ```
  658: 
  659: ## 11.4. Unsupported capability
  660: 
  661: Если routes существуют, но все отфильтрованы из-за missing capabilities:
  662: 
  663: ```text
  664: HTTP 400
  665: error.code = unsupported_capability
  666: ```
  667: 
  668: ## 11.5. No route available
  669: 
  670: Если routes существуют, но все недоступны из-за balance/cooldown/rate limit/missing key/disabled:
  671: 
  672: ```text
  673: HTTP 503
  674: error.code = no_route_available
  675: ```
  676: 
  677: ---
  678: 
  679: # 12. Reseller balance reserve
  680: 
  681: ## 12.1. Preflight reserve
  682: 
  683: Перед отправкой upstream request gateway должен зарезервировать estimated upstream cost на reseller:
  684: 
  685: ```text
  686: reseller.reserved_cents += estimated_upstream_cost_cents
  687: ```
  688: 
  689: Reserve нужен, чтобы один instance не отправил слишком много одновременных запросов в reseller, у которого расчётный баланс уже почти исчерпан.
  690: 
  691: ## 12.2. Release reserve on safe failure
  692: 
  693: Если request не был принят upstream или точно не начал обрабатываться, reserve освобождается:
  694: 
  695: ```text
  696: reseller.reserved_cents -= estimated_upstream_cost_cents
  697: ```
  698: 
  699: Safe failure examples:
  700: 
  701: ```text
  702: connection refused before response headers
  703: DNS error
  704: TLS handshake error
  705: local timeout before request write completed
  706: missing reseller API key
  707: route selection failure before upstream
  708: ```
  709: 
  710: ## 12.3. Reconcile reserve on success
  711: 
  712: После успешного upstream response gateway должен:
  713: 
  714: ```text
  715: reseller.reserved_cents -= estimated_upstream_cost_cents
  716: reseller.balance_cents -= actual_upstream_cost_cents
  717: ```
  718: 
  719: Если actual upstream cost неизвестен, используется conservative calculated cost.
  720: 
  721: ## 12.4. Actual > estimated
  722: 
  723: Если actual upstream cost больше estimated:
  724: 
  725: ```text
  726: reseller.balance_cents -= actual_upstream_cost_cents
  727: ```
  728: 
  729: Даже если reseller balance станет отрицательным, ledger должен отразить фактический расход.
  730: 
  731: Estimator должен быть conservative, чтобы такое происходило редко.
  732: 
  733: ---
  734: 
  735: # 13. Retry between routes
  736: 
  737: ## 13.1. Retry boundary
  738: 
  739: Retry на следующий route разрешён только если:
  740: 
  741: ```text
  742: upstream точно не начал обработку request
  743: или ошибка классифицирована как safe retryable before-response error
  744: ```
  745: 
  746: Retry запрещён, если есть риск двойной обработки upstream.
  747: 
  748: ## 13.2. Retryable route failures
  749: 
  750: Route может быть исключён и следующий route попробован при:
  751: 
  752: ```text
  753: rate_limit
  754: quota_exceeded
  755: insufficient_reseller_balance
  756: provider_5xx before response body is committed
  757: timeout before response headers
  758: connection_error before response headers
  759: ```
  760: 
  761: ## 13.3. Non-retryable failures
  762: 
  763: Retry запрещён при:
  764: 
  765: ```text
  766: upstream returned successful HTTP status
  767: upstream returned deterministic 4xx caused by client request
  768: response headers already received and body passthrough started
  769: request may have been processed by upstream
  770: ```
  771: 
  772: ## 13.4. Same-family retry only
  773: 
  774: Retry разрешён только внутри того же:
  775: 
  776: ```text
  777: api_family
  778: endpoint_kind
  779: client_model
  780: ```
  781: 
  782: ---
  783: 
  784: # 14. Cooldown
  785: 
  786: ## 14.1. Cooldown reasons
  787: 
  788: Route может быть переведён в cooldown по причинам:
  789: 
  790: ```text
  791: rate_limit
  792: quota_exceeded
  793: insufficient_reseller_balance
  794: provider_5xx
  795: timeout
  796: manual_disabled
  797: healthcheck_failed
  798: missing_reseller_api_key
  799: auth_error
  800: ```
  801: 
  802: ## 14.2. Cooldown durations
  803: 
  804: Cooldown durations задаются через config.
  805: 
  806: Рекомендуемые defaults:
  807: 
  808: ```text
  809: TOKENIO_ROUTE_COOLDOWN_RATE_LIMIT=60s
  810: TOKENIO_ROUTE_COOLDOWN_QUOTA_EXCEEDED=24h
  811: TOKENIO_ROUTE_COOLDOWN_5XX=30s
  812: TOKENIO_ROUTE_COOLDOWN_TIMEOUT=30s
  813: TOKENIO_ROUTE_COOLDOWN_AUTH_ERROR=24h
  814: ```
  815: 
  816: ## 14.3. Manual disabled
  817: 
  818: Если route или reseller отключён вручную, это не cooldown.
  819: 
  820: Это persistent disabled state.
  821: 
  822: ## 14.4. Cooldown visibility
  823: 
  824: Обычный client response не должен раскрывать detailed cooldown internals.
  825: 
  826: Admin API может показывать:
  827: 
  828: ```text
  829: route_id
  830: cooldown_until
  831: cooldown_reason
  832: last_error_code
  833: last_error_at
  834: ```
  835: ## 14.5. Route skip reasons
  836: 
  837: Route skip reasons are diagnostic values for route selection and admin/debug visibility.
  838: 
  839: They are not necessarily cooldown reasons.
  840: 
  841: Minimum route skip reasons:
  842: 
  843: ```text
  844: missing_reseller_api_key
  845: manual_disabled
  846: cooldown_active
  847: missing_capability
  848: insufficient_reseller_balance
  849: rate_limit_exceeded
  850: concurrency_limit_exceeded
  851: unsupported_model_rewrite_policy
  852: invalid_route_price
  853: pricing_unavailable
  854: ```
  855: 
  856: `unsupported_model_rewrite_policy` means:
  857: 
  858: ```text
  859: route requires model_rewrite_policy = provider_model,
  860: but the selected provider adapter cannot safely rewrite only model identifier for this api_family.
  861: ```
  862: 
  863: This is a configuration/adapter compatibility issue, not a temporary upstream failure.
  864: 
  865: 
  866: ---
  867: 
  868: # 15. Rate limits
  869: 
  870: ## 15.1. Route-level limits
  871: 
  872: Route может иметь limits:
  873: 
  874: ```text
  875: requests_per_minute
  876: tokens_per_minute
  877: concurrent_requests
  878: ```
  879: 
  880: Если значение равно `0`, limit считается disabled.
  881: 
  882: ## 15.2. One instance assumption
  883: 
  884: Первая версия работает в режиме:
  885: 
  886: ```text
  887: single instance
  888: ```
  889: 
  890: Поэтому in-memory limiter допустим для route-level request pacing.
  891: 
  892: Но reseller balance reserve должен храниться в Postgres, потому что это source of truth для accounting.
  893: 
  894: ## 15.3. Rate limit exceeded
  895: 
  896: Если local route limit exceeded, route временно исключается из candidates.
  897: 
  898: Если все routes исключены:
  899: 
  900: ```text
  901: HTTP 503
  902: error.code = no_route_available
  903: ```
  904: 
  905: ---
  906: 
  907: # 16. Provider adapter boundary
  908: 
  909: Provider-specific logic не должна попадать в generic route selector.
  910: 
  911: Provider-specific behavior живёт в adapters:
  912: 
  913: ```text
  914: forwarding adapter
  915: usage extractor
  916: error classifier
  917: token estimator
  918: capability mapper
  919: ```
  920: 
  921: Generic route selector не должен знать:
  922: 
  923: ```text
  924: как именно OpenRouter возвращает ошибку
  925: как именно Gemini считает usage
  926: как именно Anthropic называет поля
  927: как именно Hydra возвращает cost_request
  928: ```
  929: 
  930: Route selector работает только с нормализованными результатами adapter layer.
  931: 
  932: ---
  933: 
  934: # 17. Models endpoint interaction
  935: 
  936: `GET /v1/models` строится из route registry.
  937: 
  938: Для каждой `client_model` gateway должен определить public availability.
  939: 
  940: Если есть хотя бы один available route:
  941: 
  942: ```text
  943: active = true
  944: pricing = cheapest available route sell price
  945: capabilities = intersection of available routes for this client_model and endpoint type
  946: ```
  947: 
  948: Решение первой версии для capabilities:
  949: 
  950: ```text
  951: capabilities = intersection of available routes for this client_model and endpoint type
  952: ```
  953: 
  954: 
  955: Причина:
  956: 
  957: ```text
  958: /v1/models не должен обещать combination capabilities,
  959: если ни один concrete route не может обслужить такой request.
  960: ```
  961: 
  962: Будущая версия может вернуть `capability_profiles[]`, но первая версия использует conservative intersection.
  963: 
  964: 
  965: 
  966: Если нет available routes:
  967: 
  968: ```text
  969: active = false
  970: ```
  971: 
  972: `/v1/models` не раскрывает internal route/reseller data.
  973:

 1000: api_key_env value

 1003: ```

 1037: 
 1038: # 20. Acceptance criteria
 1039: 
 1040: Routing layer считается реализованным, если:
 1041: 
 1042: ```text
 1043: 1. Route lookup использует api_family + endpoint_kind + client_model.
 1044: 2. Unknown model возвращает 400 unknown_model.
 1045: 3. Missing capability возвращает 400 unsupported_capability.
 1046: 4. No available route возвращает 503 no_route_available.
 1047: 5. Cheapest available route выбирается по estimated upstream cost.
 1048: 6. Route в cooldown не выбирается.
 1049: 7. Disabled reseller не выбирается.
 1050: 8. Route с missing api_key_env не выбирается.
 1051: 9. Route с insufficient reseller balance не выбирается.
 1052: 10. Перед upstream request создаётся reseller reserve.
 1053: 11. Safe failure освобождает reserve.
 1054: 12. Success reconciles reserve and actual upstream cost.
 1055: 13. Retry не происходит между разными API families.
 1056: 14. Retry не происходит после unsafe upstream processing.
 1057: 15. Route with unsupported model_rewrite_policy is skipped before selection.
 1058: 16. Generic selector не содержит provider-specific if/switch по конкретным APIs.
 1059: 17. Unit tests покрывают route ordering, capability filtering, cooldown, balance filtering, missing env, unsupported_model_rewrite_policy, retry boundary и no_route_available.
 1060: ```
